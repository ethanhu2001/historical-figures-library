# Figures ground claims via Anthropic's native web search tool, scoped to Figure turns only

We considered a client-managed tool-loop against a third-party search vendor (Brave/Serper/Tavily): define a `search` function-calling tool, execute it ourselves on a `tool_use` block, round-trip the result in a second `messages.create` call. We chose Anthropic's server-side web search tool instead — passed via the `tools` argument, executed and resolved by the API in a single call — because ADR-0001 already committed to a single-provider model with no evidence of needing provider flexibility, and a vendor-agnostic search abstraction would be exactly that kind of unearned flexibility. This keeps `LLMClient` a one-shot wrapper rather than growing an agentic loop inside it.

We also scoped search to `Convener.prompt_figure` only — not `select_figures`, `choose_next_speaker`, or `synthesize`. Those three are orchestration and summarization of claims Figures already made in the debate; letting the Convener search independently would let it originate ungrounded claims at the summary stage instead of only judging which Figure-made claims held up, which is its role per the domain glossary.
